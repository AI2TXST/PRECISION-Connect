# Connect 2.0 Pipeline

This project predicts adverse outcomes for home health patients using the national OASIS dataset (~15.7M assessment rows, ~1.8M episodes) merged with ZIP-level SDOH data. Two tasks are modeled: **Task 1** predicts unplanned hospitalization among all home health patients (stable vs. hospitalized); **Task 2** predicts ED admission vs. discharge among patients who visited an emergency department during a home health episode. Both tasks use LightGBM trained with Optuna hyperparameter search on a held-out 80/10/10 train/val/test split, with threshold tuning on the validation set to optimize macro F1 against severe class imbalance (~5:1 for Task 1, ~5:1 for Task 2).

| `task2/feature_selection.py` | Task 2 | Drops comorbidity score columns, then applies variance threshold, pairwise correlation filter, NaN imputation, and target correlation filter to reach ~330 features | `data/National_Task2_ED_Cohort_encoded.parquet` | `data/National_Task2_330features.parquet` |
| `task2/eda.py` | Task 2 | EDA and baseline models (logistic regression, random forest, LightGBM) on the 330-feature dataset; generates top-20 correlation plot | `data/National_Task2_330features.parquet` | `figures/task2/top20_correlations_postFS.png`, console output |
| `task2/train_optuna.py` | Task 2 | Tunes LightGBM with Optuna on the 330-feature dataset; threshold-tunes on val set; saves best model | `data/National_Task2_330features.parquet` | `models/task2_330_best_params.pkl`, console metrics |
| `task2/eval.py` | Task 2 | Retrains LightGBM with best Optuna params, tunes classification threshold on val set, reports full test-set metrics | `data/National_Task2_330features.parquet` | `models/task2_330_best_params.pkl`, console metrics |
| `task2/sdoh_ablation.py` | Task 2 | Ablation study: trains two separate models — one on clinical features only, one on SDOH features only — to quantify each feature group's contribution | `data/National_Task2_330features.parquet` | `models/task2_clinical_only.pkl`, `models/task2_sdoh_only.pkl`, console metrics |
